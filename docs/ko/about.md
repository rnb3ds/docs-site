# CyberGo 문서 소개

## 문서 사이트

CyberGo 문서 사이트는 [CyberGoDev](https://github.com/cybergodev) 조직의 통합 기술 문서 센터로, 모든 Go 오픈소스 라이브러리에 대해 한국어, 영어, 중국어, 일본어, 러시아어 5 개 언어의 포괄적인 문서를 제공합니다.

이 사이트는 개발자가 각 라이브러리의 기능, API 및 모범 사례를 빠르게 이해하고 프로덕션 환경에서 효율적이고 안전하게 사용할 수 있도록 돕습니다.

## 문서 범위

| 프로젝트 | 설명 | GitHub |
|----------|------|--------|
| **JSON** | 고성능, 스레드 안전 JSON 처리 라이브러리 | [cybergodev/json](https://github.com/cybergodev/json) |
| **JWT** | 단 3 개의 함수로 모든 JWT 작업을 처리하는 프로덕션급 JWT 라이브러리 | [cybergodev/jwt](https://github.com/cybergodev/jwt) |
| **HTTPC** | TLS 1.2+ 및 SSRF 보호를 지원하는 최신 고성능 HTTP 클라이언트 | [cybergodev/httpc](https://github.com/cybergodev/httpc) |
| **HTML** | 지능형 기사 인식을 통한 프로덕션급 HTML 콘텐츠 추출 | [cybergodev/html](https://github.com/cybergodev/html) |
| **DD** | 초당 300 만 건 이상 처리하는 고성능 로깅 라이브러리 | [cybergodev/dd](https://github.com/cybergodev/dd) |
| **ENV** | 제로 의존성 설계의 프로덕션 준비 환경 변수 관리 라이브러리 | [cybergodev/env](https://github.com/cybergodev/env) |

## 기여하기

문서 품질 개선에 도움을 주시는 모든 형태의 기여를 환영합니다:

### 문서 편집

각 문서 페이지의 오른쪽 상단에는 "GitHub 에서 이 페이지 편집" 버튼이 있습니다. 클릭하여 직접 편집하고 Pull Request 를 제출하세요.

### 문제 보고

문서에서 오류나 부족한 점을 발견하면 다음 방법으로 피드백해주세요:

- [문서 문제 보고](https://github.com/cybergodev/docs-site/issues/new?template=doc-issue.md) — 문서 버그 또는 개선 제안 제출
- [GitHub Discussions](https://github.com/cybergodev/docs-site/discussions) — 질문, 토론 및 경험 공유

### 기여 워크플로우

1. [docs-site](https://github.com/cybergodev/docs-site) 저장소를 Fork
2. 변경 사항을 위한 새 브랜치 생성
3. 변경 내용을 설명하는 Pull Request 제출
4. 리뷰 및 병합 대기
