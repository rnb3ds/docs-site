# Оптимизация производительности

## Сравнение предустановок конфигурации

| Показатель | Default | Secure | Performance | Minimal |
|-------------|---------|--------|-------------|---------|
| Тайм-аут Request | 30с | 15с | 60с | 30с |
| MaxIdleConns | 50 | 20 | 100 | 10 |
| MaxConnsPerHost | 10 | 5 | 20 | 2 |
| MaxRetries | 3 | 1 | 3 | 0 |
| MaxResponseBodySize | 10 МБ | 5 МБ | 50 МБ | 1 МБ |
| HTTP/2 | Включён | Включён | Включён | Включён |
| Cookies | Отключены | Отключены | Включены | Отключены |
| Защита от SSRF | Включена | Включена | Включена | Включена |
| FollowRedirects | Включено | Отключено | Включено | Отключено |

## Выбор сценария

| Сценарий | Рекомендуемая предустановка | Рекомендации по настройке |
|----------|----------------------------|--------------------------|
| Общие веб-сервисы | Default | - |
| Обработка пользовательских URL | Secure | - |
| Внутренние микросервисы с высокой нагрузкой | Performance | Увеличить MaxIdleConns |
| Одноразовые скрипты | Minimal | - |
| Сервис загрузки файлов | Performance | Увеличить MaxResponseBodySize |
| Финансовые/медицинские API | Secure + пользовательская | Добавить промежуточное ПО аудита |

```go
// Высокопроизводительный сценарий
client, _ := httpc.New(httpc.PerformanceConfig())

// Тонкая настройка поверх предустановки
cfg := httpc.PerformanceConfig()
cfg.Timeouts.Request = 120 * time.Second
cfg.Connection.MaxIdleConns = 200
client, _ := httpc.New(cfg)
```

## Повторное использование пула объектов

HTTPC имеет встроенный пул объектов Result, используйте `ReleaseResult` для возврата:

```go
result, err := client.Get(url)
if err != nil {
    return err
}
defer httpc.ReleaseResult(result) // Возврат в пул объектов
```

:::tip Совет
В высоконагруженных сценариях `ReleaseResult` может значительно снизить нагрузку на GC.
:::

## Антипаттерны производительности

| Антипаттерн | Причина | Правильный подход |
|-------------|---------|-------------------|
| Создание клиента для каждого запроса | Соединения не переиспользуются | Глобальное использование клиента |
| Игнорирование ReleaseResult | Увеличение нагрузки на GC | Возврат через defer |
| Слишком большой MaxResponseBodySize | Высокое потребление памяти | Установите разумный лимит |
| Использование result.String() в горячем пути | Накладные расходы на создание строки | Используйте Body() напрямую |
