# О документации CyberGo

## Сайт документации

Сайт документации CyberGo — единый центр технической документации организации [CyberGoDev](https://github.com/cybergodev), предоставляющий подробную документацию на китайском и английском языках для всех библиотек Go с открытым исходным кодом.

Этот сайт помогает разработчикам быстро приступить к работе и глубоко понять функциональность, API и лучшие практики каждой библиотеки для эффективного и безопасного использования в продакшене.

## Охват документации

| Проект | Описание | GitHub |
|--------|----------|--------|
| **JSON** | Высокопроизводительная потокобезопасная библиотека обработки JSON | [cybergodev/json](https://github.com/cybergodev/json) |
| **JWT** | Библиотека JWT продакшен-уровня всего с 3 функциями | [cybergodev/jwt](https://github.com/cybergodev/jwt) |
| **HTTPC** | Современный высокопроизводительный HTTP-клиент с поддержкой TLS 1.2+ и защитой от SSRF | [cybergodev/httpc](https://github.com/cybergodev/httpc) |
| **HTML** | Инструмент извлечения HTML-контента продакшен-уровня с интеллектуальным распознаванием статей | [cybergodev/html](https://github.com/cybergodev/html) |
| **DD** | Высокопроизводительная библиотека логирования — более 3 млн операций/сек | [cybergodev/dd](https://github.com/cybergodev/dd) |
| **ENV** | Готовая к продакшену библиотека управления переменными окружения без зависимостей | [cybergodev/env](https://github.com/cybergodev/env) |

## Как внести вклад

Мы приветствуем любой вклад в улучшение качества документации:

### Редактирование документации

На каждой странице документации в правом верхнем углу есть кнопка «Редактировать эту страницу на GitHub». Нажмите для прямого редактирования и отправки Pull Request.

### Сообщение о проблемах

Если вы обнаружили ошибки или пробелы в документации, сообщите об этом:

- [Сообщить о проблеме в документации](https://github.com/cybergodev/docs-site/issues/new?template=doc-issue.md) — отправить сообщение об ошибке или предложение по улучшению
- [GitHub Discussions](https://github.com/cybergodev/docs-site/discussions) — задавать вопросы, обсуждать и делиться опытом

### Процесс участия

1. Сделайте Fork репозитория [docs-site](https://github.com/cybergodev/docs-site)
2. Создайте новую ветку для изменений
3. Отправьте Pull Request с описанием изменений
4. Дождитесь рецензирования и слияния
