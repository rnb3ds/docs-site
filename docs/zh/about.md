# 关于 CyberGo 文档

## 文档站点

CyberGo 文档站点是 [CyberGoDev](https://github.com/cybergodev) 组织的统一技术文档中心，为所有 Go 开源库提供中文、英文、韩文、日文、俄文五种语言的详尽文档。

本站旨在帮助开发者快速上手并深入理解每个库的功能、API 和最佳实践，确保在生产环境中高效、安全地使用这些工具。

## 文档覆盖

| 项目 | 说明 | GitHub |
|------|------|--------|
| **JSON** | 高性能、线程安全的 JSON 处理库 | [cybergodev/json](https://github.com/cybergodev/json) |
| **JWT** | 生产级 JWT 库，仅需 3 个函数即可完成所有操作 | [cybergodev/jwt](https://github.com/cybergodev/jwt) |
| **HTTPC** | 现代高性能 HTTP 客户端，支持 TLS 1.2+ 和 SSRF 防护 | [cybergodev/httpc](https://github.com/cybergodev/httpc) |
| **HTML** | 生产级 HTML 内容提取工具，支持智能文章识别 | [cybergodev/html](https://github.com/cybergodev/html) |
| **DD** | 高性能日志库，每秒处理超过 300 万次操作 | [cybergodev/dd](https://github.com/cybergodev/dd) |
| **ENV** | 生产就绪的环境变量管理库，零依赖设计 | [cybergodev/env](https://github.com/cybergodev/env) |

## 如何贡献

我们欢迎所有形式的贡献，帮助改进文档质量：

### 编辑文档

每个文档页面右上角都有"在 GitHub 编辑此页"按钮，点击即可直接编辑并提交 Pull Request。

### 报告问题

如果发现文档中的错误或不足，可以通过以下方式反馈：

- [报告文档问题](https://github.com/cybergodev/docs-site/issues/new?template=doc-issue.md) — 提交文档 Bug 或改进建议
- [GitHub Discussions](https://github.com/cybergodev/docs-site/discussions) — 提问、讨论和分享经验

### 贡献流程

1. Fork [docs-site](https://github.com/cybergodev/docs-site) 仓库
2. 创建新的分支进行修改
3. 提交 Pull Request 并描述变更内容
4. 等待审核和合并
